================================================================================
                            PROJECT OVERVIEW
================================================================================

This project is an AI-powered system that helps users check their eligibility 
for UK visas by analyzing their application details against official UK 
government visa policy documents. The system uses Retrieval-Augmented 
Generation (RAG) to provide accurate, policy-based responses.

KEY FEATURES:
✓ Interactive web-based form for 5 visa types
✓ Entity extraction from user inputs
✓ Semantic search through visa policy documents
✓ AI-powered eligibility determination
✓ Clear ELIGIBLE/NOT ELIGIBLE status display
✓ Detailed assessment with requirements and documents needed

================================================================================

================================================================================
                        DETAILED FILE DESCRIPTIONS
================================================================================

1. visa_form_app.py
   -------------------------
   PURPOSE: Main Streamlit web application
   
   FUNCTIONS:
   - load_retriever(): Loads FAISS retriever with caching
   - get_llm_client(): Creates OpenAI client for LM Studio
   - query_visa_policy(question, user_context): 
     * Retrieves relevant documents (top 5)
     * Builds context from retrieved documents
     * Calls LLM for detailed analysis
     * Makes eligibility determination
     * Returns decision and detailed assessment
   
   - build_eligibility_query(visa_type, user_data, visa_specific_data):
     * Constructs comprehensive queries based on visa type
     * Includes specific user details for better retrieval
   
   UI COMPONENTS:
   - Visa type selector (5 types)
   - Personal information form
   - Contact information form
   - Declarations section
   - Visa-specific input fields (dynamic based on selection)
   - AI query section with custom question input
   - Eligibility status display (ELIGIBLE/NOT ELIGIBLE)
   - Detailed assessment with structured sections
   - Application data summary (expandable)
   - Search query viewer (expandable)
   
   WORKFLOW:
   1. User selects visa type
   2. Form dynamically loads visa-specific fields
   3. User fills all required information
   4. User clicks "Check Eligibility"
   5. System builds comprehensive query
   6. FAISS retrieves relevant policy documents
   7. LLM analyzes documents + user data
   8. System displays ELIGIBLE/NOT ELIGIBLE status
   9. Detailed assessment shown with requirements


2. rag.py
   -------------------------
   PURPOSE: Command-line RAG interface for testing
   
   FUNCTIONS:
   - load_retriever(): Initializes FAISS retriever
   - load_llm(): Connects to LM Studio
   - answer_question(question): 
     * Retrieves top 3 documents
     * Builds context
     * Generates augmented prompt
     * Calls LLM
     * Returns answer
   
   WORKFLOW:
   1. Interactive command-line loop
   2. User enters visa-related question
   3. System retrieves relevant documents
   4. LLM generates response
   5. Answer displayed to user
   
   USAGE:
   - Testing RAG pipeline
   - Quick visa policy queries
   - Development and debugging


3. retriever.py
   -------------------------
   PURPOSE: FAISS-based semantic search retriever
   
   CLASS: VisaRetriever
   
   METHODS:
   - __init__(index_path, metadata_path, model_name):
     * Loads FAISS index
     * Loads document metadata
     * Initializes embedding model
   
   - retrieve(query, top_k, return_scores):
     * Generates query embedding
     * Searches FAISS index
     * Returns top-k relevant documents with metadata
   
   - search(query, top_k):
     * Alias for retrieve method
   
   FEATURES:
   - Semantic similarity search
   - Fast vector lookup (FAISS)
   - Metadata tracking (source files)
   - Configurable top-k results


4. create_embeddings.py
   -------------------------
   PURPOSE: Generate embeddings from visa policy documents
   
   PROCESS:
   1. Reads visa policy JSONL files
   2. Extracts text chunks
   3. Generates embeddings using Sentence Transformers
   4. Saves embeddings to .npy file
   5. Creates combined JSONL with embeddings
   
   OUTPUT:
   - visa_embeddings.npy
   - combined_visa_with_embeddings.jsonl


5. store_in_vector_db.py
   -------------------------
   PURPOSE: Create FAISS vector database
   
   PROCESS:
   1. Loads embeddings from .npy file
   2. Loads metadata from JSONL
   3. Creates FAISS index
   4. Adds vectors to index
   5. Saves index and metadata
   
   OUTPUT:
   - visa_faiss.index (FAISS vector database)
   - visa_faiss_metadata.pkl (document metadata)


6. test_rag.py
   -------------------------
   PURPOSE: Test retriever functionality
   
   TESTS:
   - FAISS index loading
   - Document retrieval
   - Query processing
   - Result display


7. test_full_rag.py
   -------------------------
   PURPOSE: Test complete RAG pipeline
   
   TESTS:
   - Retriever loading
   - LM Studio connection
   - Context building
   - LLM response generation
   - Full pipeline integration

================================================================================
                        HOW THE SYSTEM WORKS
================================================================================

END-TO-END WORKFLOW:
--------------------

1. USER INPUT STAGE
   ↓
   User accesses Streamlit app (http://localhost:8503)
   ↓
   Selects visa type (e.g., Student Visa)
   ↓
   Form dynamically displays relevant fields
   ↓
   User fills in all information:
   - Personal details (name, DOB, passport, etc.)
   - Contact information
   - Declarations
   - Visa-specific details (CAS, course info, etc.)


2. QUERY GENERATION STAGE
   ↓
   User clicks "Check Eligibility"
   ↓
   System calls build_eligibility_query()
   ↓
   Generates comprehensive query based on:
   - Visa type
   - User's specific situation
   - Key entities (CAS, course level, location, etc.)
   ↓
   Example query: "Student Visa eligibility requirements for 
   international nationals | Student Visa requirements for 
   PhD level course | Student Visa CAS requirements | 
   Student Visa financial requirements..."


3. RETRIEVAL STAGE (RAG - Retrieval)
   ↓
   Query sent to VisaRetriever
   ↓
   Sentence Transformer generates query embedding
   ↓
   FAISS searches 307 document vectors
   ↓
   Returns top 5 most relevant document chunks
   ↓
   Each chunk includes:
   - Document text
   - Source file name
   - Similarity score


4. CONTEXT BUILDING STAGE
   ↓
   System extracts text from retrieved documents
   ↓
   Builds structured context:
   --- Document 1 ---
   Source: student_visa_policy.pdf
   [Policy text...]
   
   --- Document 2 ---
   Source: student_visa_policy.pdf
   [Policy text...]
   ↓
   Adds user application details
   ↓
   Creates augmented prompt combining:
   - Visa policy context (retrieved documents)
   - User application details
   - Specific question


5. LLM ANALYSIS STAGE (RAG - Generation)
   ↓
   Augmented prompt sent to LLaMA 3.2 3B
   ↓
   LLM analyzes:
   - Policy requirements
   - User's provided information
   - Match between requirements and user data
   ↓
   Generates detailed assessment with:
   - Eligibility Requirements
   - Financial Requirements
   - Required Documents
   - Fees and Timeline
   - User-specific assessment


6. ELIGIBILITY DETERMINATION STAGE
   ↓
   Second LLM call for final decision
   ↓
   Analyzes detailed assessment + user data
   ↓
   Returns one of:
   - ELIGIBLE: [reason]
   - NOT ELIGIBLE: [reason]
   - INSUFFICIENT INFORMATION: [missing items]


7. DISPLAY STAGE
   ↓
   System displays results:
   ┌─────────────────────────────────────┐
   │ ✅ ELIGIBILITY STATUS: ELIGIBLE     │
   │ Reason: Meets all requirements...   │
   └─────────────────────────────────────┘
   ↓
   Shows detailed assessment
   ↓
   Lists specific checks (CAS ✓, Funds ✓)
   ↓
   Provides expandable sections:
   - Application data summary
   - Search query used

================================================================================
                        MODEL SPECIFICATIONS
================================================================================

EMBEDDING MODEL:
----------------
Name: all-MiniLM-L6-v2
Type: Sentence Transformer
Developer: sentence-transformers
Dimensions: 384
Purpose: Convert text to semantic vectors
Use Case: Query and document embedding
Performance: Fast inference, good semantic understanding
Download: Automatic via sentence-transformers library


LANGUAGE MODEL:
---------------
Name: LLaMA 3.2 3B Instruct
Type: Large Language Model
Developer: Meta AI
Parameters: 3 billion
Hosting: LM Studio (local)
API: OpenAI-compatible
Endpoint: http://localhost:1234/v1
Temperature: 0.2 (factual, low creativity)
Max Tokens: 150 (eligibility) / Unlimited (detailed)
Purpose: Policy analysis and eligibility determination
Strengths: Instruction following, factual responses


VECTOR DATABASE:
----------------
Technology: FAISS (Facebook AI Similarity Search)
Index Type: Flat L2 (exact nearest neighbor)
Vectors: 307 document chunks
Dimension: 384 (matching embedding model)
Search Time: ~1-2ms per query
Storage: visa_faiss.index (~500KB)
Metadata: visa_faiss_metadata.pkl (~200KB)
